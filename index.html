That time I started writing an IRC bouncer in C

Recently, I have been learning about networking programming. I am also interested in programming with C, so natually, I brought the two together. This isn't the first time I have attempted to do this, but this time I mean it.

It helps me focus if I have a project to work on, especially one that means something to me. In this case, I am implementing an IRC bouncer. The idea is that I want a persistent connection to an IRC server, but I also want to attach to that persistent connection from any IRC client. I normally just use irssi with a tmux session on a remote server, and that works great except for when I am on my phone.

Building the proxy will be straightforward. What will need some thought is how a user will interface with the proxy from the client. It must be possible to manage multiple connections to multiple servers, and be able to differentiate between them. It also must be possible to retrieve history so that the user doesn't miss any messages, especially mentions.

One way this can be done is by adding commands that conform to the IRC protocol that only the proxy understands. Another option would be to have a bot that can accept specialized commands. For the messaging backfilling, the bot could reply with correct IRC replies, as if the messages had come in from the IRC server proper.

I know softare that does this already exists, but that takes all the fun out of it. My primary goal is to learn C and network programming. I mostly care about IRC as a means to that goal. Iit also helps that I use IRC every day, so I am extra motivated to use what I am building.
